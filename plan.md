# 小学全科AI学习平台 — 项目计划与技术指南

## 项目愿景

构建一个**免费、开源**的小学全科在线学习平台，利用AI从人教版（统编版）教材中提取知识体系，按学科特点自动生成**单元小测、综合考试和知识点全解**，让每一个小学生都能在语文、数学、英语、科学、道德与法治等核心学科上获得系统化的练习资源。

> **范围**：本项目聚焦"可被结构化练习"的学科。音乐、美术、体育、书法等以实践/欣赏为主的学科暂不纳入题库生成，未来可作为知识介绍模块。

---

## 一、数据源

### 教材来源

- **GitHub仓库**：[TapXWorld/ChinaTextbook · 小学](https://github.com/TapXWorld/ChinaTextbook/tree/master/%E5%B0%8F%E5%AD%A6)
- **首选版本**：**人教版**（数学、英语、科学）；语文、道德与法治采用国家统编版（即"部编版"，与人教版同属人教社出版）
- **覆盖范围**：一年级至六年级，上下册
- **格式**：PDF原件

### 学科优先级与覆盖

| 优先级 | 学科 | 版本 | 说明 |
|--------|------|------|------|
| P0 | 数学 | 人教版 | 已有Pipeline基础，最先跑通 |
| P0 | 语文 | 统编版 | 用户量最大，必须覆盖 |
| P0 | 英语 | 人教版（PEP，三年级起） | 一二年级无英语 |
| P1 | 科学 | 人教版（一年级起） | 实验/观察类内容多，提取难度较高 |
| P1 | 道德与法治 | 统编版 | 情境/案例类，题型独特 |
| P2 | 体育、音乐、美术、书法 | — | 暂不生成题库，未来作为知识介绍 |

### 教材清单（示例）

不同学科册数不同。数学/语文/道法每年级上下册共12册；英语三年级起共8册；科学一般每年级上下册共12册（具体以仓库实际为准，Pipeline以遍历目录方式发现教材，不写死清单）。

### 版权策略

教材PDF为人教社/教育部正式出版物，直接搬运原文存在版权风险。我们的策略：

1. **只提取知识体系和大纲结构**，不搬运教材原文
2. **所有题目由AI原创生成**，基于知识点而非教材原题
3. **知识点讲解用自己的语言概括**，不照抄教材表述
4. **古诗文、课文等已进入公有领域或属于教材选篇的内容**，仅引用必要片段做题干，不全文照搬
5. 平台定位为教辅工具，不替代教材本身

## 二、技术架构

### AI Pipeline（后端数据生成）

```
GitHub PDF Repo（按学科目录遍历）
       │
       ▼
  下载所有学科PDF
       │
       ▼
  pypdf 拆分页面（每10页一批，图多学科可调小）
       │
       ▼
  Gemini 3.1 Flash Lite（逐批提取，注入"学科+年级"上下文）
  PDF原件 → 结构化Markdown
       │
       ▼
  合并为完整教材Markdown
       │
       ▼
  Qwen 3.6 Plus + Structured Output（按学科选用不同schema）
  Markdown → 大纲JSON（单元 + 知识点树）
       │
       ▼
  Qwen 3.6 Plus + Structured Output（逐单元，按学科选用不同prompt）
  大纲 → 题目JSON + 知识点全解Markdown
       │
       ▼
  质量校验（学科特定规则）
       │
       ▼
  输出到 output/<学科>/ 目录
```

### 模型选择理由

| 任务 | 模型 | 理由 |
|------|------|------|
| PDF内容提取 | `google/gemini-3.1-flash-lite-preview` | 多模态能力强，能处理数学图形、英语插图、科学实验示意图、语文课文版式 |
| 大纲生成 | `qwen/qwen3.6-plus` | 中文理解强，支持structured output |
| 题目生成 | `qwen/qwen3.6-plus` | 中文命题质量高，英文题目也能胜任小学PEP水平 |

### 学科差异如何在Pipeline中体现

学科不是"换个目录"那么简单。同一份Pipeline代码通过**学科配置（subject config）** 来切换行为：

```python
SUBJECT_CONFIG = {
  "math":    { "outline_schema": MATH_OUTLINE,    "quiz_prompt": MATH_QUIZ_PROMPT,    "validators": [...] },
  "chinese": { "outline_schema": CHINESE_OUTLINE, "quiz_prompt": CHINESE_QUIZ_PROMPT, "validators": [...] },
  "english": { "outline_schema": ENGLISH_OUTLINE, "quiz_prompt": ENGLISH_QUIZ_PROMPT, "validators": [...] },
  "science": { ... },
  "ethics":  { ... },
}
```

每个学科有独立的：知识点字段定义、题型枚举、难度评分锚点、prompt模板、校验规则。

## 三、各学科侧重与题目设计

每个学科的知识结构和评估方式差别很大。**通用题型不能套用**——下面是各学科的设计原则。

### 1. 数学（P0）

**知识维度**：数与代数、图形与几何、统计与概率、综合实践

**核心题型**：
- 口算、竖式计算
- 填空、选择、判断
- 画图（简单几何）、连线/匹配
- 应用题（一步、两步、多步）
- 操作题、规律探索

**侧重**：
- 计算准确性 + 解题过程展示
- 应用题情境贴近小学生生活
- 几何题用文字+坐标描述图形，前端可考虑用SVG渲染
- 每道计算题在 `explanation` 中**必须包含逆运算验算**

**数值范围按年级约束**（一年级20以内→六年级分数四则运算）

### 2. 语文（P0）

**知识维度**：拼音、识字与写字、词语、句子、古诗文、阅读理解、口语交际、习作

**核心题型**：
- **拼音题**：看拼音写词语、给汉字注音、声调判断
- **字词题**：组词、近义词反义词、多音字、形近字辨析、查字典
- **句子题**：扩句缩句、修改病句、关联词、修辞手法
- **古诗文**：默写填空、诗句理解、作者朝代
- **课文理解**：根据课文片段回答问题（片段需为可合理引用的简短引用）
- **阅读理解**：AI原创短文 + 配套问题（不依赖教材原文）
- **看图写话/小练笔**：开放题，提供评分要点而非标准答案

**侧重**：
- **字音字形是低年级核心**，一二年级题目大量为拼音/识字
- 阅读理解的短文必须AI原创，避免版权问题
- 古诗文使用公有领域作品（教材选篇绝大多数为古代诗文）
- 作文类题目不要求标准答案，只给出**评分维度**（立意、结构、语言、字数）
- explanation 要解释"为什么选这个"，而不是只说答案

**特殊字段**：题目JSON中加入 `pinyin` 字段（用于注音题）、`chinese_chars` 字段（用于汉字相关题）

### 3. 英语（P0，三年级起）

**知识维度**：词汇、句型、语法、对话、阅读、语音

**核心题型**：
- **词汇题**：单词拼写、中英互译、单词分类
- **语音题**：选出划线部分发音不同的单词、字母组合发音
- **句型题**：句型转换（陈述↔疑问、单数↔复数）、连词成句、根据答语写问题
- **语法题**：选择填空（时态、be动词、人称代词、形容词比较级等）
- **对话题**：补全对话、情境对话排序
- **阅读理解**：AI原创短文 + 选择/判断/简答

**侧重**：
- **严格控制词汇范围**：只使用对应年级PEP教材已学单词，超纲词必须在题干给中文注释
- **句型简单**：低年级避免复合句、被动语态等
- **不做听力题**：纯文本平台无法播放音频，听力部分跳过（未来可加TTS）
- 题干和选项的英文必须语法正确、地道
- explanation 用中文解释，便于学生理解

**特殊字段**：`vocabulary_level`（标注本题涉及的核心单词）、`grammar_point`（语法点）

### 4. 科学（P1）

**知识维度**：物质科学、生命科学、地球与宇宙、技术与工程

**核心题型**：
- **概念辨析**：选择、判断（如"下列属于昆虫的是"）
- **填空题**：科学事实记忆
- **实验题**：实验步骤排序、根据现象推断结论、设计简单实验
- **观察记录**：根据图表/数据回答问题
- **简答题**：解释自然现象（"为什么白天看不到星星？"）

**侧重**：
- **重过程、重探究**：题目要体现"猜想—实验—结论"思路，而非死记硬背
- 实验题用文字描述实验装置，必要时让前端用示意图
- 避免超出小学认知的精确数值（如不要求记住引力常数）
- common_mistakes 要包含**前科学概念误区**（如"植物只通过根吸收水分"）

**特殊字段**：`experiment_required`（是否需要实验背景）、`science_domain`（学科分支）

### 5. 道德与法治（P1）

**知识维度**：自我认识、家庭生活、学校生活、社会参与、国家意识、法律常识

**核心题型**：
- **情境判断题**："小明遇到XX情况，他应该怎么做？"（选择/判断）
- **案例分析题**：给一段简短情境，分析对错、说出理由
- **观点辨析**：判断某种说法是否正确，并说明原因
- **角色扮演题**：开放题，给评分要点

**侧重**：
- **正向价值观引导**：答案不能模糊处理是非
- 情境必须贴近小学生真实生活（同学冲突、家庭分工、网络使用、公共秩序）
- 法律常识题严格按现行法律和教材表述，不臆造
- 避免说教腔，用故事化的题干
- explanation 要"讲道理"而不是"贴标签"

**特殊字段**：`scenario`（情境描述，独立于题干）、`value_dimension`（涉及的价值观维度）

## 四、数据格式设计

### 输出文件结构

```
output/
├── math/                                  # 数学
│   ├── pdfs/
│   ├── markdown/
│   ├── outlines/
│   ├── quizzes/
│   └── knowledge/
├── chinese/                               # 语文
│   └── ...
├── english/                               # 英语
│   └── ...
├── science/                               # 科学
│   └── ...
└── ethics/                                # 道德与法治
    └── ...
```

每个学科子目录内部结构与原数学Pipeline一致（pdfs / markdown / outlines / quizzes / knowledge）。

### 通用大纲JSON Schema

```json
{
  "subject": "math",
  "subject_name": "数学",
  "version": "人教版",
  "textbook": "三年级上册",
  "units": [
    {
      "unit_number": 1,
      "title": "时、分、秒",
      "knowledge_points": [
        {
          "name": "秒的认识",
          "description": "认识时间单位'秒'，知道1分=60秒",
          "difficulty": 2,
          "question_types": ["填空", "判断", "选择", "应用题"]
        }
      ]
    }
  ]
}
```

学科特定字段在 `knowledge_points[].extra` 中扩展，例如：
- 语文：`{"text_type": "古诗", "author": "李白"}`
- 英语：`{"vocabulary": ["apple", "banana"], "grammar_point": "be动词"}`
- 科学：`{"science_domain": "生命科学", "experiment_required": true}`

### 通用题目JSON Schema

```json
{
  "subject": "chinese",
  "subject_name": "语文",
  "version": "统编版",
  "textbook": "三年级上册",
  "unit": "第一单元 美丽的祖国",
  "unit_number": 1,
  "unit_test": {
    "title": "第1单元 单元小测",
    "total_score": 100,
    "time_minutes": 40,
    "questions": [
      {
        "id": 1,
        "type": "pinyin_to_word",
        "score": 4,
        "difficulty": 1,
        "knowledge_point": "本单元生字",
        "question": "看拼音写词语：mínɡ liànɡ",
        "answer": "明亮",
        "explanation": "'明'指光线充足，'亮'指有光，组合表示光线充足。",
        "extra": { "pinyin": "míng liàng", "chinese_chars": ["明", "亮"] }
      }
    ]
  },
  "exam": { "...": "..." },
  "knowledge_summary": [ "..." ]
}
```

**`type` 字段是学科特定的枚举**，每个学科有自己的题型集合，前端按 type 选择渲染组件。

### 难度评级标准（通用5级，学科锚点不同）

| 等级 | 含义 | 数学示例 | 语文示例 | 英语示例 |
|------|------|----------|----------|----------|
| 1 | 识记 | 认识图形名称 | 认读生字 | 拼写已学单词 |
| 2 | 理解 | 理解乘法含义 | 解释词语意思 | 理解简单句子 |
| 3 | 应用 | 用竖式计算 | 用词造句 | 句型转换 |
| 4 | 分析 | 两步应用题 | 段落理解 | 完形填空 |
| 5 | 创造 | 设计购物方案 | 看图写话 | 看图写句 |

### 数学公式与拼音规范

- 数学公式：LaTeX，前端用KaTeX渲染（`$1\text{分} = 60\text{秒}$`）
- 拼音：使用带声调的Unicode字符（`míng liàng`），前端不需特殊渲染
- 英语：纯文本，注意大小写和标点

## 五、Prompt工程设计

### 设计原则（通用）

1. **角色锚定**：每学科prompt设定专属角色（人教版数学教研员 / 统编语文特级教师 / PEP英语命题组 / 小学科学教研员 / 道法课程专家）
2. **学段约束**：明确告知年级上下限（词汇/数值/认知水平），防止越界
3. **structured output**：所有JSON输出走 `json_schema + strict: true`
4. **few-shot**：每个学科prompt内嵌2-3个高质量样例题
5. **学科红线**：每个prompt写明该学科的"绝不能做"（数学不算错、语文不出错别字、英语不出语法错、科学不传谣、道法不模糊是非）

### 五套学科Prompt（每套包含 大纲prompt + 题目prompt）

每个学科独立维护：
- `prompts/math.py`
- `prompts/chinese.py`
- `prompts/english.py`
- `prompts/science.py`
- `prompts/ethics.py`

PDF提取prompt（Gemini）相对通用，但会注入学科上下文，让模型知道"现在在提取一本三年级语文教材"，避免把古诗当成数学公式之类的错乱。

### 难度分布约束（按学科可调）

| 学科 | 单元小测分布 | 综合测验分布 |
|------|--------------|--------------|
| 数学 | 基础70% / 中等20% / 提高10% | 基础50% / 中等30% / 提高20% |
| 语文 | 基础60% / 中等30% / 提高10% | 基础40% / 中等40% / 提高20% |
| 英语 | 基础70% / 中等25% / 提高5% | 基础50% / 中等35% / 提高15% |
| 科学 | 基础60% / 中等30% / 提高10% | 基础50% / 中等30% / 提高20% |
| 道法 | 基础50% / 中等40% / 提高10% | 基础40% / 中等40% / 提高20% |

## 六、质量保障体系

### 自动化校验（按学科扩展）

**通用规则**（所有学科）：
- 分数总和等于 total_score
- difficulty 在 1-5 之间
- answer / explanation 非空
- type 在该学科枚举内

**学科特定规则**：

| 学科 | 特定校验 |
|------|----------|
| 数学 | 计算题 explanation 必须包含验算步骤；数值范围匹配年级 |
| 语文 | 拼音题 extra.pinyin 字段必填；汉字字符在 GB2312 范围内 |
| 英语 | 题干和答案的英文做基础拼写校验；超纲词检测（对照PEP词表） |
| 科学 | explanation 不出现伪科学关键词列表；实验题必须有装置/步骤描述 |
| 道法 | 答案不能为"视情况而定"等模糊表述；情境必须有明确是非判断 |

校验未通过的单元在JSON中标记 `_validation_issues` 字段，Pipeline结束时汇总报告。

### 人工审核流程

每个学科至少邀请一位**对应学科的小学老师**做抽样审核：

1. 每册随机抽5道题，审核答案正确性 + 难度匹配
2. 检查语言是否符合小学生认知
3. 道法学科额外审核价值观导向
4. 英语学科额外审核词汇是否超纲

### 未来可加的验证

- AI交叉验证（用第二个模型独立解题）
- 跨册重复题检测
- 英语词汇超纲自动统计

## 七、运行指南

### 环境准备

```bash
pip install openai requests pypdf
export OPENROUTER_API_KEY="sk-or-v1-xxxxxxxxxxxx"
```

### 文件结构

```
project/
├── prompts/
│   ├── __init__.py
│   ├── common.py        # PDF提取prompt + 通用工具
│   ├── math.py
│   ├── chinese.py
│   ├── english.py
│   ├── science.py
│   └── ethics.py
├── pipeline.py          # 主Pipeline，按学科调度
├── validators/
│   ├── common.py
│   └── <subject>.py
└── output/
```

### 运行命令

```bash
# 跑全部学科全部教材
python pipeline.py

# 只跑某一学科
python pipeline.py --subject math
python pipeline.py --subject chinese

# 只跑一本试水
python pipeline.py --subject math --only "三年级上册"

# 多学科并行（不同学科独立缓存，可并行）
python pipeline.py --subject math &
python pipeline.py --subject chinese &

# 调整每批页数
python pipeline.py --subject science --batch-size 5

# 预览
python pipeline.py --dry-run
```

### 断点续跑

每个学科独立缓存。已存在的批次markdown / 大纲JSON / 单元题目JSON 自动跳过。删除对应文件即可重新生成。

### 预估成本（粗略，5学科全量）

| 步骤 | 调用次数（5学科）| 备注 |
|------|------------------|------|
| PDF提取（Gemini） | ~600-800 | 教材册数最多的是数学和语文 |
| 大纲生成（Qwen） | ~50 | 每册1次 |
| 题目生成（Qwen） | ~300-400 | 每单元1次 |

建议先用 `--subject math --only "三年级上册"` 跑一本估算单价。

## 八、前端网站规划（下一阶段）

### 技术选型

| 层 | 技术 | 理由 |
|----|------|------|
| 框架 | Next.js / React | SSG静态生成，免费托管 |
| 样式 | Tailwind CSS | 快速开发，响应式 |
| 公式渲染 | KaTeX | 数学公式 |
| 托管 | Vercel / GitHub Pages | 免费 |
| 数据 | 静态JSON文件 | 无需后端 |

### 核心导航层级

```
首页
 └─ 学科选择（数学 / 语文 / 英语 / 科学 / 道法）
     └─ 年级选择（一到六年级，英语三起）
         └─ 上下册
             └─ 单元列表
                 ├─ 知识点全解
                 ├─ 单元小测
                 └─ 综合测验
```

### 答题组件按学科分化

**通用组件**：选择、判断、填空、简答

**学科专用组件**：
- 数学：竖式计算输入、画图题（占位/SVG）
- 语文：拼音输入、田字格汉字输入、古诗默写
- 英语：单词拼写、句型转换
- 科学：实验步骤拖拽排序
- 道法：情境卡片 + 选择

前端按 `question.type` 字段动态选择渲染组件。

## 十、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| AI生成的答案错误 | 严重影响信任 | 验算prompt + validators + 人工抽检 + 未来交叉验证 |
| 学科特定错误（语文错别字、英语语法错、道法价值观偏差） | 严重 | 学科专属validator + 学科老师审核 |
| 难度不匹配年级 | 体验差 | prompt中按年级强约束 + 老师审核 |
| 教材版权 | 法律风险 | 只提取知识体系，题目原创，不搬运原文，课文阅读自创短文 |
| API成本超预期 | 预算压力 | 学科级缓存 + 单本试水估算 |
| 模型下线/涨价 | Pipeline中断 | 模型配置集中管理 |
| 学科prompt质量不齐 | 部分学科题目差 | 每学科独立迭代，先发布质量过关的学科 |

## 十一、关键决策记录

| 决策 | 选择 | 替代方案 | 理由 |
|------|------|----------|------|
| 学科范围 | 语数英科道5科 | 全10科 / 仅数学 | 体音美书法不适合题库化；5科覆盖核心学习需求 |
| 优先版本 | 人教版 / 统编版 | 多版本并行 | 用户最广，先做一版打磨质量 |
| Pipeline结构 | 单Pipeline + 学科配置 | 每学科一份脚本 | 复用基础设施，学科差异通过配置注入 |
| 题型设计 | 学科独立枚举 | 通用题型 | 学科差异巨大，强行统一会丢失学科特性 |
| PDF提取 | Gemini读原件 | pypdf文本提取 | 教材图形/版式信息丰富，纯文本丢失太多 |
| 学科推进顺序 | 数学先行 | 全部并行 | 数学Pipeline已验证，先用它打磨基础设施 |
| 英语听力 | 暂不做 | TTS生成 | MVP阶段聚焦文本，未来再加 |
| 语文阅读理解短文 | AI原创 | 引用教材课文 | 规避版权，AI原创短文质量已足够 |
| JSON格式保障 | structured output | prompt约束 | API层面强制更可靠 |
| 前端托管 | 静态站 | 带后端 | MVP免费托管，无需后端 |
